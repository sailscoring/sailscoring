## Backlog Conventions

### Item ID Format

- `F-XXX` — Feature work
- `B-XXX` — Bug fix
- `T-XXX` — Technical debt
- `R-XXX` — Research/spike

### Estimation

_Define estimation approach (story points, t-shirt sizes, etc.) if used._

### Prioritization Criteria

1. MVP requirements first
2. User value
3. Technical dependencies
4. Risk reduction

### Definition of Ready

Before an item enters "Ready for Development":

- [ ] Clear description of what to build
- [ ] Acceptance criteria defined
- [ ] Dependencies identified
- [ ] No blocking questions

### Definition of Done

For an item to be marked complete:

- [ ] Code complete
- [ ] Tests passing
- [ ] Code reviewed (if applicable)
- [ ] Acceptance criteria verified
- [ ] Documentation updated (if applicable)
